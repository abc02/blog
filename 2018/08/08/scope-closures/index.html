作用域

  • JavaScript 动态 or 解释执行语言
  • 编译原理
    • a
      • 传统编译语言的流程中 -> 分词/词法分析 -> 解析/语法分析 -> 代码生产 经历三个步骤统称”编译”
        1. 词法分析 -> 字符串分解成代码块 -> 词法单元
        2. 词法单元流（数组） -> 元素逐级嵌套所组成 -> 抽象语法树（AST）
          • var a = 2;抽象语法树 -> VariableDeclaration顶级节点 -> Identifier子节点 & AssignmentExpression子节点 -> NumericLiteral子节点
        3. AST -> 转换可执行代码
    • b
      1. JavaScript引擎 -> 负责整个程序编译及执行过程
      2. 编译器 -> 负责语法分析及代码生产
      3. 作用域 -> 负收集并维护又所有声明的标示符组成的一系列查询，实施一套严格规则，确定当前执行代码对这些标示符的访问权限
    • c
      • 编译器 -> 语法分析 -> 代码生成 -> 引擎执行 -> 作用域协助查询
      • LHS查询
        • 试图找到变量的容器本身
        • 查找的目的是对变量进行赋值
      • RHS查询
        • retrieve his source value 取到它的源值
        • 目的是获取变量的值
  • 作用域嵌套
    • 当前作用域 -> 上一级作用域 -> 全局作用域
  • 异常
    • ReferenceError
      • 同作用域判别失败相关
      • RHS查询不到所有嵌套作用域所需变量
      • LHS
        • 严格模式禁止自动或隐式创建全局变量
        • 宽松/懒惰模式会全局作用域中创建变量
    • TypeError
      • 代表作用域判别成功了，但对结果的操作是非法或不合理
      • RHS
        • 对变量的值进行不合理的操作
          • 例如对非函数类型的值进行函数调用
          • 引用null或undefined类型的值中的属性

词法作用域

  • 工作模式
    • 词法作用域
    • 动态作用域
  • 词法阶段
    • 词法作用域就是你书写代码时函数声明的位置来决定的
    • 大部分标准语言编译器第一工作阶段叫做词法化
      - 词法化的过程对源代码中的字符进行检查
- 如果有状态的解析过程还会赋予单词语义
- 该概念是理解词法作用域及其名称来历的基础
    • 词法欺骗 不推荐使用
      • eval
      • with
